# 链群服务激励扩展协议（初稿）

## 1. 关于协议

此扩展协议，基于 LOVE20 扩展协议标准，扩展行动参与机制，对链群行动中的链群服务者做激励

- 链上参与：参与链群服务激励行动
- 自动计算：链上自动计算铸币激励
- 激励分配：支持设置多个接收地址和分配比例

## 2. 初始参数

- 代币地址
- 链群行动 id
- 最大激励分配地址数

## 3. 激励铸造

链群服务激励 = 链群服务行动总激励 × 链群服务者已服务的所有链群铸币激励 / 所有链群铸币激励总和

## 4. 激励分配

链群服务激励可以设置二次分配，在铸造激励的时候，自动分配到设置的所有地址，剩余部分默认为链群服务者的激励。

### 4.1 分配设置

链群服务者可以在行动阶段，设置多个接收地址和分配基点（basis points）：

- 基点范围：0-10000（10000 = 100%）
- 基点总和不能超过 10000
- 接收地址数不可超过设定的上限
- 剩余部分（10000 - 基点总和）归链群服务者所有
- 同一阶段可多次设置，以最后一次设置为准
- 若当轮未设置则向前查找最近的设置
- 链群服务者在领取激励时，自动按照本轮分配设置分配激励

### 4.2 历史记录

可按轮次查询每轮最终生效的设置及激励分配明细

### 4.3 分配计算

```
某接收地址的激励 = 总激励 × 该地址基点 / 10000
链群服务者的激励 = 总激励 - 所有接收地址激励之和
```
